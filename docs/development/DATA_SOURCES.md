# 📊 三數據源驗證系統 - 數據源詳解

## 🎯 核心策略

**每支股票必須有 3 個有效數據源**

系統會自動從 11 個數據源池中智能選擇，確保：
1. 每支股票都有 3 個獨立數據源
2. 如果某個源失敗，自動切換備用源
3. 數據交叉驗證，標記異常值

## 🔄 智能分配邏輯

系統會自動：

1. **優先使用高額度源**
   ```
   Yahoo (無限) → IEX (1,666) → Finnhub (1,440) → ...
   ```

2. **負載均衡**
   ```
   均勻分配請求到各數據源，避免單一源耗盡
   ```

3. **自動容錯**
   ```
   數據源 A 失敗 → 自動切換到數據源 B → 再切換到 C
   ```

4. **額度監控**
   ```
   實時追蹤各源剩餘額度
   接近上限時自動跳過
   保留 10% 緩衝
   ```

## 🔧 故障排除

### Q: 某個數據源一直失敗？

A: 系統會自動跳過，使用備用源。無需擔心。

### Q: 額度用完了怎麼辦？

A: 系統會自動檢測並切換到其他源。建議隔天再跑。

### Q: 數據不一致怎麼辦？

A: 系統會標記異常，並在報告中顯示「數據差異」警告。
