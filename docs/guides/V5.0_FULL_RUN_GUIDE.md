# v5.0 完整可執行版 - 使用指南

v5.0 已接上**真實市場數據**與**真實 LLM**，可完整執行每日分析與決策。

**與 v4.3 一致**：
- **數據網**：與 v4.3 相同（DataFetcher / yfinance，可擴展多數據源）
- **LLM**：與 v4.3 相同四供應商（Scitely, Cohere, Mistral, OpenRouter），沿用同一組 .env
- **Key 不足時**：一 LLM 兼多角（依 fallback 順序用第一個有 key 的），**框架不變**

## 1. 真實市場數據（與 v4.3 相同數據網）

- **來源**：與 v4.3 相同（`utils.data_fetcher.DataFetcher` / yfinance）
- **預設股票**：`AAPL, GOOGL, MSFT, TSLA, NVDA`（小樣本，最多 20 支）
- **天數**：90 天歷史（供型態與指標計算）

### 自訂掃描列表

**方式一：環境變數**

```bash
export V5_SCAN_TICKERS="AAPL,GOOGL,MSFT,TSLA,NVDA,AMD,META"
python main_v5.py --daily
```

**方式二：config.yaml**

在 `config.yaml` 的 `mvp` 下加入：

```yaml
mvp:
  scan_tickers: ["AAPL", "GOOGL", "MSFT", "TSLA", "NVDA", "AMD", "META"]
```

## 2. 真實 LLM（決策引擎）

決策引擎會呼叫 LLM 做「買/賣/持有」等決策。需至少設定一組 API Key。

### 四供應商（與 v4.3 完全相同）

| 變數 | 角色 | 預設模型 |
|------|------|----------|
| `SCITELY_API_KEY` | 基本面 | qwen3-235b-a22b-instruct |
| `COHERE_API_KEY` | 技術面 | command-a-03-2025 |
| `MISTRAL_API_KEY` | 風險 | mistral-small-latest |
| `OPENROUTER_API_KEY` | 宏觀 | meta-llama/llama-3.1-8b-instruct |

Key 不足時：依 fallback 順序（Scitely → Cohere → Mistral → OpenRouter）用**第一個有 key 的**，一 LLM 兼多角，框架不變。

### 設定方式（與 v4.3 共用 .env，四 Key 沿用）

v5.0 會載入**與 v4.3 相同的 .env**。  
若你已在 v4.3 設好 **SCITELY / COHERE / MISTRAL / OPENROUTER** 任一或多個，**無需再設**，v5.0 直接沿用：

```bash
# v4.3 Stage 3 已用的四個變數，v5.0 沿用（任一即可運行，多個則四角齊全）
SCITELY_API_KEY=...
COHERE_API_KEY=...
MISTRAL_API_KEY=...
OPENROUTER_API_KEY=...

# 可選：指定模型
SCITELY_MODEL=qwen3-235b-a22b-instruct
OPENROUTER_MODEL=meta-llama/llama-3.1-8b-instruct
```

- 只有一個 Key：啟動時顯示「✓ LLM 已連接（1 個 Key，兼多角）：xxx」
- 四個都有：顯示「✓ LLM 已連接（四供應商與 v4.3 相同）：scitely, cohere, mistral, openrouter」

### 啟動時確認

執行 `python main_v5.py --daily` 時：

- 若有設定 API Key，會顯示：**✓ LLM 已連接（決策將使用真實 API）**
- 若未設定，會顯示：**ℹ️ LLM 未配置（請設 OPENROUTER_API_KEY 或 SCITELY_API_KEY）**

未配置時仍會跑完整流程（含真實數據與型態分析），但決策為保守預設，報告會出現「LLM響應格式錯誤」；設定 API Key 後即可得到真實 LLM 決策。

## 3. 執行流程（完整版）

```bash
cd /Users/lautinyam/stock_scanner

# 1. 建議先設好 .env 中的 API Key（見上）

# 2. 執行每日分析（會拉真實數據 + 若已設 Key 則呼叫 LLM）
python main_v5.py --daily
```

流程會：

1. **拉取真實 OHLCV**：對掃描列表中的股票取 90 天資料  
2. **圖表型態識別**：用真實 K 線做突破等型態偵測  
3. **因果推理 / 情緒 / Multi-Agent**：依現有模組與資料運算  
4. **決策引擎**：若有 LLM，會用防幻覺機制呼叫 API 並解析決策 JSON  
5. **輸出報告**：寫入 `reports/daily/` 並在終端顯示摘要

## 5. 常見問題

**Q: 為什麼報告還是「今日無操作建議」？**  
A: 若未設 `OPENROUTER_API_KEY` 或 `SCITELY_API_KEY`，決策引擎不會呼叫 LLM，會採用保守預設（無操作）。請在 `.env` 設定其中一個 Key 後再跑一次。

**Q: 可以只用真實數據、不接 LLM 嗎？**  
A: 可以。不設 API Key 時，仍會拉真實數據並跑型態等分析，僅決策為保守預設。

**Q: 掃描更多股票會變慢嗎？**  
A: 會。目前單次最多 20 支（可在程式內調整），建議先用小樣本（5–10 支）確認流程與 LLM 決策正常後再擴大。
