# REISHI 霊視 v4.3 → v5.0 回測變更與注意事項（給 Orchestrator / 回測同事）

> **對象**：負責回測與 Orchestrator 的同事  
> **範圍**：從 v4.3 到目前 v5.0 為止，所有與回測相關的更改與注意事項  
> **最後更新**：依專案最新狀態

---

## 一、一句話總結

- **介面契約不變**：`config.json` 輸入、`backtest_summary.csv` / `backtest_trades.csv` 輸出格式與欄位**完全一致**，Orchestrator 讀取邏輯**無需修改**。
- **有兩套回測入口**：**v4.3（main.py）** 是完整 pipeline 逐日回測；**v5.0（main_v5.py）** 是獨立回測模組、簡化版引擎。依需求選用。
- **初始資金**：兩者皆預設 **40,000 HKD**，且皆可由 `config.json` 覆寫。

---

## 二、版本與回測入口對照

| 項目 | v4.3 | v5.0（迄今） |
|------|------|--------------|
| **主程式** | `main.py` | `main_v5.py` |
| **回測觸發** | `--backtest`（1 個或 2 個日期） | `--backtest`（固定 2 個日期） |
| **回測邏輯** | 完整 pipeline：Stage 1→2→3 + 持倉監控 + 每日決策，逐日跑 | 獨立模組 `core/backtest_engine.py`，簡化模擬 |
| **config.json** | 啟動時 `apply_orchestrator_config()` 讀取並覆寫全域 | 回測時由 `BacktestEngine` 讀取，僅影響回測參數 |
| **輸出目錄** | `reports/backtest_range/START_to_END/` | `reports/backtest_range/START_to_END/`（相同） |
| **輸出檔** | 同上目錄下 `backtest_summary.csv`、`backtest_trades.csv` | 同上，欄位一致 |
| **初始資金** | 40,000 HKD（可由 config.json 改） | 40,000 HKD（可由 config.json 改） |

## 三、v4.3 回測（main.py）— 完整 pipeline

### 3.1 指令

```bash
# 單日回測（只跑該日一次）
python main.py --backtest 2025-01-10

# 區間回測（每個交易日跑完整 pipeline）
python main.py --backtest 2025-01-01 2025-01-31

# 區間 + 小股數加快（每日約 15 檔掃描）
python main.py --backtest 2025-01-01 2025-01-31 --quick
```

### 3.2 行為摘要

- 啟動時會呼叫 **`apply_orchestrator_config()`**，若專案根目錄有 **`config.json`** 會覆寫：  
  `stage1_weights`、`stage2_weights`、`backtest_initial_cash`、`backtest_pct_per_new_entry`、`backtest_add_pct`、`backtest_reduce_pct`。
- 區間回測：對區間內**每個交易日**跑一次 **Stage 1 → Stage 2 → Stage 3**，再跑 **daily_monitor**（持倉監控 + 當日決策），決策僅用「前一交易日及之前」的數據（無偷看）；執行價為**當日收盤價**（由 `backtest_simulator.apply_decision` + `daily_monitor.get_current_price` 等，多數據源）。
- 輸出寫入：`reports/backtest_range/START_to_END/backtest_summary.csv`、`backtest_trades.csv`；區間內每日報告在 `reports/backtest_range/START_to_END/YYYY-MM-DD/`。

### 3.3 注意事項

- 區間回測耗時長（每日都跑完整 Stage 1→2→3 + LLM），適合「要與實盤邏輯一致」的嚴謹回測。
- 依賴 **config.py** 的 `BACKTEST_INITIAL_CASH` 等（可被 config.json 覆寫）、**backtest_simulator**、**daily_monitor**、多數據源（如 `utils.data_sources`）及 LLM（Stage 3）。

## 四、v5.0 回測（main_v5.py）— 獨立模組

### 4.1 指令

```bash
# 僅支援「開始日 + 結束日」兩個參數
python main_v5.py --backtest 2025-01-01 2025-01-31
```

- **不支援**單日 `--backtest 2025-01-10` 這種單一日期寫法；一定要給兩個日期。

### 4.2 行為摘要

- 回測時**不會**載入 v5.0 的每日分析、監控、LLM 決策等完整流程，只跑 **`core/backtest_engine.run_backtest_for_orchestrator()`**。
- **config.json** 由 **BacktestEngine** 在回測時讀取（預設 `config.json`），只影響回測：  
  `backtest_initial_cash`、`backtest_pct_per_new_entry`、`backtest_add_pct`、`backtest_reduce_pct`、`stage1_weights`、`stage2_weights`（目前引擎為簡化版，權重可讀但未必全部參與模擬）。
- 當前 v5.0 回測引擎為**簡化模擬**（依日期迴圈與預設候選列表），**沒有**真正逐日跑 Stage 1→2→3 或 v5.0 的 AI 決策鏈。
- 輸出目錄與檔名：**與 v4.3 相同**  
  `reports/backtest_range/START_to_END/backtest_summary.csv`、`backtest_trades.csv`。

### 4.3 注意事項

- 若你需要「與實盤/完整選股邏輯一致」的歷史回測，應以 **v4.3（main.py）** 為主。
- v5.0 回測適合：快速產出**格式正確**的 CSV 給 Orchestrator、驗證介面與評分流程；或日後在 v5.0 內逐步接上真實決策邏輯時再對齊。

## 五、介面契約（兩版共用，Orchestrator 無需改）

### 5.1 輸入：config.json（專案根目錄）

| 參數 | 說明 | 預設 |
|------|------|------|
| `backtest_initial_cash` | 初始資金（HKD） | 40000.0 |
| `backtest_pct_per_new_entry` | 每筆新買入佔現金比例 | 0.20 |
| `backtest_add_pct` | 加碼時動用現金比例 | 0.10 |
| `backtest_reduce_pct` | 減碼時賣出持倉比例 | 0.50 |
| `stage1_weights` | Stage 1 技術指標權重（v4.3 會用） | 見 config.json.example |
| `stage2_weights` | Stage 2 權重（v4.3 會用） | 見 config.json.example |

- v4.3：啟動時就套用，影響整次執行。  
- v5.0：僅回測時讀取，只影響 BacktestEngine。

### 5.2 輸出：backtest_summary.csv（每日盈虧）

| 欄位 | 說明 |
|------|------|
| date | 交易日 YYYY-MM-DD |
| portfolio_value | 組合總價值（HKD） |
| cash | 剩餘現金（HKD） |
| positions_count | 持倉檔數 |
| return_pct | 累計報酬率（%） |

### 5.3 輸出：backtest_trades.csv（交易明細）

| 欄位 | 說明 |
|------|------|
| date | 交易日 |
| ticker | 股票代碼 |
| action | buy / sell / add / reduce |
| price | 執行價格 |
| quantity | 股數 |

- **兩版輸出欄位與語意一致**，Orchestrator 可同一套程式讀取。

## 七、重要注意事項清單（給回測同事）

1. **介面不變**  
   config.json 結構與兩個 CSV 的欄位、語意在 v4.3 與 v5.0 保持一致，**現有 Orchestrator 程式不必改**。

2. **選對入口**  
   - 要**完整 pipeline、逐日 Stage 1→2→3 + 持倉監控 + 當日決策**：用 **v4.3**  
     `python main.py --backtest 2025-01-01 2025-01-31`（可加 `--quick`）。  
   - 要**快速產出格式正確的 CSV** 或未來接 v5.0 決策：用 **v5.0**  
     `python main_v5.py --backtest 2025-01-01 2025-01-31`。

3. **v4.3 回測參數**  
   - 單日：`--backtest 2025-01-10`。  
   - 區間：`--backtest 2025-01-01 2025-01-31`，可加 `--quick` 減少每日掃描數以加快。

4. **v5.0 回測參數**  
   - 僅區間：`--backtest START END`（**一定要兩個日期**），無 `--quick`。

5. **初始資金**  
   - 兩版預設皆 **40,000 HKD**。  
   - 若要改：在專案根目錄放 `config.json`，設定 `backtest_initial_cash`；v4.3 在 main 啟動時套用，v5.0 在回測時由 BacktestEngine 讀取。

6. **評分與成本**  
   - Scanner（v4.3 / v5.0）**只產出原始 CSV**；Sortino、MDD、交易成本、過擬合檢測等均由 **Orchestrator** 計算。

7. **無偷看**  
   - v4.3 區間回測：決策僅用「前一交易日及之前」的數據，執行價為當日收盤價。  
   - v5.0 回測引擎為簡化版，邏輯上仍以「當日及之前」為準，但未跑完整選股與 LLM。

## 八、變更歷程摘要（v4.3 → v5.0 與後續）

| 版本/更新 | 與回測相關的變更 |
|-----------|------------------|
| v4.3 | 完整回測：Stage 1→2→3 + daily_monitor，區間逐日、config.json、backtest_summary/trades、40k HKD、多數據源、NYSE 交易日曆。 |
| v5.0.0 | 新增 v5.0 架構（每日分析、決策引擎等），**未含回測**。 |
| v5.0.1 | 新增 **core/backtest_engine.py** 與 **main_v5.py --backtest START END**；輸出格式與 Orchestrator 契約一致；回測為獨立模組，不影響 --daily/--monitor/--stats。 |
| 後續 | v5.0 接真實市場數據與 LLM（main_v5 每日分析）；回測介面與 CSV 格式維持不變；若未來 v5.0 回測改為「逐日跑 v5.0 決策」，會在同一輸出契約下擴充。 |
