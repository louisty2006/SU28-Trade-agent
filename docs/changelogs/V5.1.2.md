# REISHI v5.1.2

小 patch：修正數據管理「下載中斷後續跑」行為並加入除錯日誌，方便排查問題。

---

## 這個 patch 更新了什麼

### 1. 下載續跑路徑一致

- **問題**：在數據管理選 [B] 下載 20 年 K 線時，若中途 Ctrl+C，下次再跑有時會從 0 重新開始，而不是從上次 partial 進度繼續。
- **修正**：每次開始下載／修復該年時先呼叫 `_resolve_data_roots()`，讓 `MARKET_STOCKS_DIR`、`out_path` 一律依「當前工作目錄」計算，避免專案根與執行目錄不一致導致寫入與讀取路徑不同。

### 2. 運行證據日誌（方便排查）

- **repair_start**：進入該年下載時列印一次，包含 `project_root`、`cwd`、`out_path`、`out_path_exists`，可確認路徑與是否偵測到既有 partial。
- **write_partial_ok / write_partial_fail**：每次寫入 partial 檔成功或失敗時列印，含寫入路徑、股票數、錯誤訊息（若失敗），可確認是否有真的寫出檔案。
- **resume_load_ok / resume_load_fail**：下次執行時若讀取既有 parquet 成功或失敗會列印，含路徑與錯誤訊息，可確認讀取階段是否正常。

上述日誌會寫入 `.cursor/debug.log` 與專案根 `debug_run.log`，且關鍵 tag 會同時列印到終端（stdout），即使 log 檔寫不進去也能從終端輸出看到。

### 3. 版本號

- 主選單 banner 與數據管理畫面由 v5.1 改為 **v5.1.2**。
