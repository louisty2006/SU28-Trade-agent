# REISHI v5.2

版本 5.2 整合 5.1～5.1.2 的數據管理改進，並加入狀態表即時更新與依賴說明。

---

## 更新摘要

### 1. 數據管理：下載與續跑

- **單一資料夾、按年存檔**：所有 K 線寫入 `data/market_data/us_stocks/`，每年一個檔案 `YYYY.parquet`。中斷後再選 [B] 或該年修復會從既有 partial 繼續，不會另建資料夾。
- **路徑一致**：下載／讀取前會 `_resolve_data_roots()`，依當前工作目錄決定路徑，回測「本地數據」與數據管理使用同一目錄。
- **Parquet 引擎**：啟動下載前檢查 `pyarrow` 或 `fastparquet`；若皆未安裝會提示並中止，避免下載後無法寫入。`requirements.txt` 已加入 `pyarrow`。
- **[B] 倒序下載（2025→2005）**：優先完成近年數據，早年（≤2009）達 30% 即跳過，加快整體進度。近年資料較齊，成功率高。
- **[B] 倒序下載（2025→2005）**：優先完成近年數據，早年（≤2009）達 30% 即跳過，加快整體進度。近年資料較齊，成功率高。

### 2. 狀態表即時更新

- 進入「數據管理」時，**K 線狀態表**會依目前磁碟上的 parquet 即時顯示（上次已下載會立刻反映）。
- 每次下載或修復（[F]、[1]–[9]、[A]、[B]、[C]、[D]、[E]）完成後，會**自動重新整理並列印**「年份／狀態／股票數／完整度」表格，無需重進選單。
- 選 [B] 下載 20 年時，**每完成一年**會印一次更新表格，全部完成後再印一次。

### 3. 中斷與續跑（Ctrl+C）

- **Ctrl+C** 時會先寫入目前進度到該年 parquet，再顯示：`已存妥：YYYY.parquet 共 N 檔 X MB，下次續跑會從此繼續。`
- 若中斷時**尚未有新的 50 檔寫入**（例如剛開始並行、尚未有任一 worker 完成），數字會與上次相同，屬正常；可等進度條或「已寫入 partial」出現後再中斷，以確保有新進度被存檔。
- **驗證方式**：執行 [B] → 等至少一次「已寫入 partial（X 檔）」或進度條有變化 → Ctrl+C → 再執行 [B]，應看到「續跑… 已有 X 檔」且 X 比上次大。

### 4. 運行日誌（除錯用）

- `repair_start` / `write_partial_ok` / `write_partial_fail` / `resume_load_ok` / `resume_load_fail` 會寫入 `.cursor/debug.log` 與 `debug_run.log`，關鍵 tag 會列印到終端，方便排查續跑或寫入問題。

### 5. 依賴與專案設定

- **requirements.txt**：新增 `pyarrow>=14.0.0`（Parquet 讀寫）。
- **.gitignore**：新增 `data/market_data/`，本地下載的 parquet 不納入版控。
