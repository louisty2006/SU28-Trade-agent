# REISHI v5.1 優化完成摘要

依 Agent 任務 Prompt 完成「補基本面、情緒用新聞+LLM、Multi-Agent 按 AlphaRock 類 format」三大項與整合。

---

## 改動與新增檔案

### 新增
- **`analysis/fundamental_analysis.py`**
  - `FundamentalResult`：PE、PB、ROE、revenue_growth、earnings_growth、profit_margin、sector、industry、market_cap、summary_text
  - `FundamentalAnalyzer.analyze_batch(tickers, as_of_date=None, backtest_start=None, on_ticker=None)`：即時用 `get_yahoo_info` + `extract_financial_data`；回測用 `get_yahoo_financials_as_of`

### 改寫
- **`analysis/sentiment_analysis.py`**
  - `analyze_news(ticker, news_list)` 使用 `news_list` 內容組文，呼叫 `core.llm_clients.LLMClients`，要求 JSON：score(-1~1)、confidence、key_factors、risks
  - 無新聞或 LLM 失敗時回退中性（score=0.5，標註原因）
  - `analyze_batch` 介面不變，主流程傳入的 `news_by_ticker` 已使用

- **`analysis/multi_agent.py`**
  - 三角色：**Fundamental**（Scitely）、**Sentiment**（Mistral）、**Valuation/技術**（Cohere），各用 LLM 產出 `AgentAnalysis`
  - 單輪共識（OpenRouter）產出 `consensus_action`、`consensus_score`、`disagreements`、`final_recommendation`
  - `analyze_all(candidates, data)` 的 `data` 須含 `market_data`、`fundamental_by_ticker`、`sentiment_by_ticker`
  - 回傳 `{"by_ticker": {ticker: MultiAgentResult}, "summary": "..."}`

### 修改
- **`core/decision_engine.py`**
  - `AllAnalyses` 新增可選欄位 `fundamental`
  - `to_dict()` 與 `summary()` 納入 `fundamental` 與 `multi_agent`（當為 dict 時格式化 `by_ticker` 的共識與分歧）

- **`main_v5.py`**
  - 匯入並初始化 `FundamentalAnalyzer`
  - **run_daily**：在「數據獲取與驗證」之後新增步驟 **[2/9] 基本面分析**，後續步驟改為 [3/9]～[9/9]；`AllAnalyses` 傳入 `fundamental=fundamental_by_ticker`；`multi_agent.analyze_all` 傳入 `data={ market_data, fundamental_by_ticker, sentiment_by_ticker }`
  - **run_daily_for_backtest**：在步驟 1 之後新增「步驟1b 基本面分析」（使用 `as_of_date`、`backtest_start`）；`AllAnalyses` 與 `analyze_all` 同上

---

## 主流程呼叫順序（優化後）

1. 數據獲取與驗證  
2. **基本面分析**（新增）  
3. 圖表型態識別  
4. 因果推理  
5. 情緒分析（新聞 + LLM）  
6. Multi-Agent 協作（三角色 + 共識，使用基本面與情緒）  
7. 霊視記憶參考  
8. 決策引擎  
9. 驗證與審計
